* Welcome
  :PROPERTIES:
  :VISIBILITY: children
  :END:

  One keybinding to rule them all: =SPC SPC=.

  Press it anywhere.  dwim reads the context — cursor position, active
  region, major mode, project, installed packages — and offers only the
  actions that make sense /right now/.  No prefix maps to memorise.
  No sub-menus to navigate.  Just press and read.

  This buffer is itself an Org file, so every feature described here is
  /live/: code blocks run, links open, checkboxes tick.  Treat it as a
  playground.

* Try it now
  :PROPERTIES:
  :VISIBILITY: children
  :END:

  Work through these steps in order. Each one is a real Emacs interaction, not a
  description of one. Use TAB to cycle trees and blocks visibility

** Step 1 — invoke DWIM in this buffer

   Press =SPC SPC= right now (point can be anywhere).

   You are in an Org buffer, so you will see categories like =Org=, =Note=, =Org
   Agenda=, =Consult=, =Buffer=, and more. Type a few letters to narrow; press =RET= to
   run the highlighted action, =C-g= to cancel. For instance, go to the line below
   and 'SPC SPC checkbox' or 'SPC SPC otc'

   - [ ] I pressed =SPC SPC= and saw the candidate menu.

** Step 2 — run a code block

   Place point inside the block below and press =C-c C-c= to evaluate
   it.  The result appears immediately beneath it.

   #+BEGIN_SRC emacs-lisp
   (format "Hello from dwimacs!  Today is %s, Emacs %s."
           (format-time-string "%A %d %B %Y %H:%M:%S")

   #+END_SRC

   Now press =SPC SPC= with point /still inside the block/.  Notice the

   - [ ] I ran the block with =C-c C-c=.
   - [ ] I pressed =SPC SPC= inside the block and ran it with a =Babel= action.

** Step 3 — select a region, then DWIM

   Select any word or sentence in this buffer (=v= in evil normal state,
   or =C-SPC= + motion), then press =SPC SPC=.

   The =Region= provider fires and surfaces actions like upcase,
   downcase, sort, comment, fill, eval, and pipe through a shell
   command — all scoped to your selection.

   - [ ] I selected text and saw region-specific actions in the menu.

** Step 4 — open the debug log

   =SPC SPC explain= collects all actions and writes a log of
   every provider that ran, what it returned, and what was rejected and
   why.  Useful for understanding why an action is or is not appearing.

   #+BEGIN_SRC emacs-lisp
   (init-dwim-explain)
   #+END_SRC

   - [ ] I opened the debug log and read through it.

* How DWIM works
  :PROPERTIES:
  :VISIBILITY: folded
  :END:

  When you press =SPC SPC=:

  1. Every function in =init-dwim-providers= is called with no arguments.
  2. Each *provider* inspects the current context and returns a list of
     action plists, or =nil= if nothing applies.
  3. Actions are merged, deduplicated, and sorted by =:priority=.
  4. The sorted list is presented via Vertico + Orderless.  Marginalia
     annotates each candidate with its description.
  5. You pick one; dwim calls its =:action= function.

  A broken provider is isolated — it never prevents other providers from
  showing their actions.

** The action anatomy

   Every action is a plist with these keys:

   #+BEGIN_SRC emacs-lisp :eval never
   (init-dwim-make-action
    :title       "My action"          ; shown in the menu
    :description "What it does"       ; Marginalia annotation
    :category    "MyCategory"         ; grouping label
    :priority    80                   ; higher = listed first
    :predicate   (lambda () …)        ; return non-nil to include
    :action      (lambda () …))       ; called on selection
   #+END_SRC

** Useful meta-commands

   | Command                    | What it does                          |
   |----------------------------+---------------------------------------|
   | =SPC SPC=                    | The DWIM menu                         |
   | =SPC SPC explain=            | Debug log: offered / rejected actions |

* Evil & editing
  :PROPERTIES:
  :VISIBILITY: folded
  :END:

  Evil runs with =SPC= as leader in normal, visual, motion, and emacs
  states.  Emacs state also binds =hjkl= for navigation so you never
  need to switch states just to move.

** Motions and operators

   All standard evil motions and operators work: =d=, =c=, =y=, =>=, =<=…
   evil-surround adds =ys=, =cs=, =ds= for surrounding pairs.

** Region actions

   Mark a region, then =SPC SPC=.  The =Region= provider offers:
   upcase, downcase, titlecase, =sort-lines=, =align-regexp=,
   =fill-region=, comment/uncomment, eval, and pipe through a shell
   command.  The =Expand-region= provider grows or shrinks the selection
   by semantic unit.

** Numbers at point

   With point on a number, DWIM offers increment, decrement, and
   conversions between decimal, hex, and binary.

   #+BEGIN_SRC emacs-lisp
   ;; Put point on the number below, then press SPC SPC.
   42
   #+END_SRC

** Rectangles

   In Evil, visual state (=v=) defines an Emacs rectangle using the columns of the
   first and last positions of the selection across the spanned lines. =SPC SPC=
   then offers insert, delete, string-replace, copy, and yank for the marked
   rectangle.

* Completion & snippets
  :PROPERTIES:
  :VISIBILITY: folded
  :END:

  Corfu provides the in-buffer popup backed by Cape sources:
  =cape-file=, =cape-dabbrev=, and LSP completions when Eglot is active.
  The popup appears automatically; =TAB= / =S-TAB= cycle candidates.

  Yasnippet + yasnippet-snippets handle snippet expansion.
  =SPC SPC= → =Snippet=:

  - /Expand snippet at point/
  - /Insert snippet/ — browse all snippets for this mode
  - /New snippet/ — create one for this mode
  - /Reload snippets/ — pick up edits without restarting

* Files, buffers & windows

** Buffers

   open in Dired, copy file path.

   winner-redo.

   SPC= =Dired= for mark, copy, move, chmod, compress.

   keep-all, next/previous conflict.

* Tips & tricks
  :PROPERTIES:
  :VISIBILITY: folded
  :END:

  - *Narrow the menu fast* — type fragments in any order.  =lsp rd=
    finds =LSP: Rename definition=.  Initialism: =idf= → =Insert
    defun=.

  - *Disable a noisy provider* — add to =init-dwim-disabled-providers=:
    #+BEGIN_SRC emacs-lisp :eval never
    (add-to-list 'init-dwim-disabled-providers 'init-dwim-calc-provider)
    #+END_SRC

  - *Visual undo* — =SPC SPC= → =Undo= → /Visual undo tree/ (or
    =M-x vundo=) shows the full branching undo history as a tree.

  - *which-key* pops up after 0.5 s on any partial chord, revealing
    all continuations.

  - *winner-mode* remembers window layouts: =C-c <left>= / =C-c <right>=
    step through history.

  - *Focus mode* — =SPC SPC= → =Focus= toggles writeroom-mode +
    olivetti-mode (body width 88) for distraction-free writing.

  - *Persistence* — recentf, savehist, and save-place are on: files,
    minibuffer history, and cursor positions survive restarts.

  - *Emacs state hjkl* — in emacs state =h j k l= still move the
    cursor, so you never need to switch to normal state just to navigate.
